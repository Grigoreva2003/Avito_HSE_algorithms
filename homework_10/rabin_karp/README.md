# Rabin-Karp

## Задача
Реализовать поиск подстроки в строке с помощью алгоритма Рабина-Карпа. 

К решению нужно прикрепить отчет с

* объяснением rolling hash
* оценкой сложности итогового алгоритма

Тесты продолжаем писать.

## Rolling hash (полиномиальный хеш)
Хеш считается как 
$$
h(s) = (s\_0 b^{m-1} + s\_1 b^{m-2} + ... + s_{m-1}) \\ b \mod (p)
$$
 где $b$ — основание (например, 257), $p$ — большое простое. Для окна длины m обновляем хеш за O(1): вычитаем вклад ушедшего символа, домножаем на b, прибавляем новый символ.

## Сложность
* Время: O(n + m) в среднем (каждое окно обновляется за O(1)); O(nm) только при множественных коллизиях.
* Память: O(1) дополнительной (не считая результата).
